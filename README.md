An enterprise-grade, AI-powered decision support system built on the **London Smart Meter Dataset**.  
It forecasts electricity consumption, detects abnormal usage patterns, segments households by behavior, and delivers personalized energy-saving recommendations — all visualized through a modern interactive dashboard.

## 🎯 Project Highlights

| Capability | Technology Used |
|---|---|
| ⚡ Energy Forecasting | XGBoost, Random Forest, Linear Regression |
| 🚨 Anomaly Detection | Isolation Forest + Z-Score |
| 🏠 Household Segmentation | K-Means Clustering |
| 💡 Savings Recommendations | Rule-Based Engine (ToU Tariff-Aware) |
| 📊 Interactive Dashboard | Streamlit + Plotly |

## 🚀 Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```
> Requires Python 3.10+

### 2. Generate synthetic data (no download needed)

```bash
python src/generate_sample_data.py
```

### 3. Run the data pipeline

```bash
python src/data_pipeline.py
```

This executes the ETL process: data loading → cleaning → feature engineering → validation.

### 4. Launch the dashboard

```bash
streamlit run dashboard/app.py
```

Open **http://localhost:8501** in your browser. ✅

## 🔬 Machine Learning Pipeline

| Phase | Module | What it does | Key Decision |
|---|---|---|---|
| 1–2 | `data_pipeline.py` | Load CSVs, interpolate gaps, engineer lag/rolling/calendar features | Max 4-step gap interpolation to avoid faking data |
| 3 | `forecasting.py` | XGBoost + Random Forest + Linear Regression, evaluated on MAE/RMSE/MAPE/R² | Chronological train/test split — no random shuffle for time-series |
| 4 | `anomaly_detection.py` | Z-score (univariate) + Isolation Forest (multivariate) → combined severity score | No labeled anomalies available; two-method ensemble for confidence |
| 5 | `clustering.py` | K-Means on each household's 48-slot average daily load shape | Labels are relative to dataset quantiles — stays meaningful at any scale |
| 6 | `recommendations.py` | Maps cluster + anomaly data → £/month savings + plain-language advice | Rule-based and explainable — appropriate for stakeholder demonstrations |

## 🌐 Using the Real London Dataset

1. Download from Kaggle: [Smart Meters in London](https://www.kaggle.com/datasets/jeanmidev/smart-meters-in-london)
2. Unzip and place block files:
```
data/
  halfhourly_dataset/
    block_0.csv
    block_1.csv
    ...
  informations_households.csv
```
3. Start with a small subset:
```python
# In src/data_pipeline.py
run_pipeline(n_blocks=10)   # load only 10 blocks (~few hundred households)
```
4. Scale up gradually once the end-to-end flow is verified.

> ⚠️ **Remove synthetic files** before mixing in real data.

## 💷 Tariff Assumptions

The recommendation engine uses **Low Carbon London dynamic Time-of-Use tariffs**:

| Period | Hours | Rate |
|---|---|---|
| 🔴 Peak | 16:00–19:00 | 67.20p / kWh |
| 🟢 Off-Peak | 00:00–07:00 | 3.99p / kWh |
| 🟡 Normal | All other hours | 11.76p / kWh |

Update `PEAK_RATE`, `OFFPEAK_RATE`, `NORMAL_RATE` in `src/recommendations.py` to match your tariff structure.
